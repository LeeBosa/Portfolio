React의 기본

React란?

React는 현재 현업에서 인기 있는 웹/앱의 View를 개발할 수 있도록 하는 자바스크립트 라이브러리이다.

보통 우리가 생각하는 어플리케이션(웹, 앱 혹은 데스크톱용 소프트웨어)을 만들기 위해서는 사용자가 조작하기 위한 UI(User Interface), UI를 컨트롤 하기 위한 로직, 데이터를 처리하는 비즈니스 로직 등 3가지 부분으로 개발이 필요하다.

이렇게 특정 부분을 나누어 개발하는 방법론을 * MVC 패턴, * MVVM패턴 이라고 하며, React.js는 View 즉, 사용자가 조작하기 위한 UI를 만드는 것을 도와주는 라이브러리이다.

* MVC(Model-View-Controller) 패턴

웹/앱을 모델(Model), 뷰(View), 컨트롤러(Controller) 3개의 영역으로 분할하고 각 구성 요소에게 고유한 역할을 부여하는 개발 방식입니다.

모델(Model) : Data, 정보의 가공을 책임지는 컴포넌트

뷰(View) : 사용자에게 보여지는 부분, 즉 유저 인터페이스(User Interface)를 의미

컨트롤러(Controller) : 모델(Model)과 뷰(View) 사이를 이어주는 브릿지(Bridge) 역할

* MVVM(Model-View-View Model) 패턴

MVC 패턴에서 Controller를 빼고 View Model을 추가한 패턴. View와 Model이 서로 전혀 알지 못하기 때문에 독립성을 유지할 수 있고, 독립성을 유지하기 때문에 효율적인 유닛 테스트가 가능한 패턴입니다.

뷰 모델(View Model) : 뷰(View)를 표현하기 위해 만들어진 뷰(View)를 위한 모델(Model)

React의 특징

1. React는 선언형(Declarative)이다.

2. React는 컴포넌트 기반으로 재사용성(Reusability)이 뛰어나다.

3. React는 가상돔(Virtual DOM)기반으로 가볍다.

4. React 컴포넌트는 state와 props를 가진다.

우리는 여기에서 다음의 키워드를 얻을 수 있다. 선언형(Declarative), 컴포넌트(Component), 재사용성(Reusable), 가상 돔(Virtual DOM).

React - 선언형(Declarative)

소프트웨어 공학에서 자주 접하게 되는 개념은 패러다임(Paradigm)이다. 즉, 프로그래밍을 어떻게 할 것인가에 대한 논의가 활발하게 이루어진다.

이 중 React를 하면 자주 접할 수 있는 패러다임이 선언형(Declarative) 프로그래밍과 명령형(Imperative) 프로그래밍이다.

React는 선언형(Declarative) 성격에 맞게 컴포넌트(Component)를 얻기 위해 <tags></tags> jsx 문법을 통해 구현한다. 즉, jsx를 얻기 위한 알고리즘에 대한 구현을 하지 않는다. (예를 들어, document.createElement나 해당 컴포넌트(Component)의 변경사항을 체크하는 알고리즘, 리-렌더링 여부에 대한 알고리즘을 구현하지 않는다.)

이와 같은 선언형(Declarative) 특성은, React를 사용할 때 화면 설계라는 초첨에 맞춰서 개발할 수 있도록 해주므로, 다른 부분에 대한 고민을 최소화 해주어 높은 생산성을 보장할 수 있도록 해준다.

Component Based Development (CBD)

우리는 컴포넌트(Component) 단위로 개발을 해야한다.

컴포넌트(Component)는 독립적인 단위의 소프트웨어 모듈을 말한다. 즉, 소프트웨어를 독립적인 하나의 부품으로 만드는 방법이다. React는 웹에서 쓰는 각 요소들을 컴포넌트로 만들 수 있게 해 기존의 UI를 다른 화면에서 다시 쓰거나, 다른 프로젝트에서 다시 쓸 수 있도록 하는 장점(높은 재사용성)을 가진다.

1. 독립적인 코드 블럭 (HTML + CSS + Javascript)

2. 중복되는 키워드를 하나의 컴포넌트로 표현하고 싶다. (이것을 이룬게 React다.)

3. 작업의 단위 = 컴포넌트

컴포넌트(Component)?

HTML 엘리먼트(Element)와 컴포넌트(Component)를 비교해서 살펴보자. 먼저, <img src="이미지주소"/>처럼 컴포넌트(Component)는 내가 지은 이름으로 태그(tag)를 만들 수 있다.

img태그에서 src를 설정해준 것처럼 컴포넌트(Component)에서 name을 Mark라고 설정해주었다. Mark라는 이름(Name)은 컴포넌트(Component)를 태그 안에서 표현할 때 사용된다. prop은 HTML에서의 어트리뷰트(Attribute)를 컴포넌트(Component)에서 표현할 때 사용하는 단어이다. prop으로 표현할 태그가 여러 개 있을 경우 대신 props를 사용한다. 또 외부에서 <내가 지은 이름1> 태그 안으로 어떤 데이터를 넣어주고 싶을 때도 prop을 사용한다.

<button>버튼</button> 태그에 주목해보자. 버튼 태그 사이에 넣은 텍스트는 실제 웹사이트에 표현될 내용이다. 컴포넌트(Component)는 children이라는 prop을 통해 태그 사이에 넣을 텍스트를 불러올 수 있다.
{prop={false}, children="내용"}

이 객체 이름이 props이다.

사용자 입장에서는 HTML element와 컴포넌트는 크게 다르지 않다. 그러나 <img> 혹은 <button>은 이미 어떻게 동작할지 구현되어 있지만, <내가 지은 이름1> 혹은 <내가 지은 이름2>는 어떻게 동작할지 구현되지 않았다. 그렇다면 우리가 할 일은, 즉 컴포넌트 개발자가 해야 할 일은 <내가 지은 이름1>이 props를 받아서 그 안에서 어떻게 표현되고 어떻게 동작할지를 구현하는 것이다.

가상 돔(Virtual DOM)

Virtual DOM(가상 돔)이란 어딘가에 데이터로만 존재하는 것(눈에 보이지 않는다). 가상 돔이 하는 역할은 가상의 트리 구조를 만들고 이를 실제로 브라우저에 그리는 것이다.

따라서 React란 가상 돔을 만들고 이를 실제로 브라우저에 그리도록 도와주는 언어라고 할 수 있다.

가상 돔과 진짜 돔을 구분하는 가장 쉬운 방법은, State(이벤트 트리거)가 변경됐을 때, 개발자 도구로 찍어서 요소가 보이면 진짜 돔이고, 보이지 않으면 가상 돔이다.

Why Virtual DOM?

돔을 직접 제어하지 않는 가상 돔은, 우리가 표현(이럴 땐 A, 저럴 땐 B를 렌더링 해라)하기만 하면 알아서 상태를 비교해서 자동으로 렌더링 해준다. 이 경우 기존 HTML + CSS + JS 기반으로 한 진짜 돔과 비교했을 때 메모리 사용량을 줄이고, 보안성을 높이는 효과를 준다.

가상 돔은 State(이벤트 트리거)가 변했을 때, 가상 돔에 렌더링한 후 기존의 DOM과 달라진 부분만 변경한다.

이 방식은 이전 UI 상태를 메모리에 유지한 체, 변경될 UI의 최소집합을 계산하여 필요한 부분만 교체하여 불필요한 업데이트를 줄이고, 렌더링 속도를 올려준다.

CSR VS SSR

CSR(Client Side Rendering)

JS가 전부 다운로드 되어 리액트 어플리케이션이 정상 실행되기 전까지 화면이 보이지 않는다.
JS가 전부 다운로드 되어 리액트 어플리케이션이 정상 실행된 후, 화면이 보이면서 유저가 인터렉션이 가능하다.

1. 서버가 빈 HTML을 response한다. (화면에 아무것도 보이지 않는다.)
2. HTML안에 들어있는 JS파일을 서버로부터 다운로드한다.
3. 브라우저가 다운로드된 JS파일을 실행한다. (JS안에 React 라이브러리가 포함되어 있다.)
4. React 실행 직후에 현재 경로에 해당하는 페이지를 렌더링 (화면이 보이고 인터렉션 가능한 상태)

SSR(Server Side Rendering)

JS가 전부 다운로드 되지 않아도, 일단 화면은 보이지만 유저가 사용할 수 없다.
JS가 전부 다운로드 되어 리액트 어플리케이션이 정상 실행된 후, 유저가 사용가능하다.

1. 서버에서 문자열로 표현된 HTML을 미리 받아온다.
2. 이 HTML을 클라이언트가 받는 순간 화면에 문서가 나타난다.
3. HTML을 받으면서 안에 있는 JS파일을 받아온다 (사용할 수는 없는 단계).
4. React를 실행한다. (이 때도 사용할 수는 없다.)
5. React가 실행된 후 인터렉션이 가능하다.

render() 메서드

리액트를 사용하려면 두 개의 모듈(라이브러리)이 필요하다. 1번 React을 통해 컴포넌트를 생성할 수 있고, 만든 컴포넌트를 페이지에 렌더링하는 걸 도와주는게 2번 ReactDOM이다.

ReactDOM에서 가장 많이 쓰는 함수는 render라는 메서드이다. render는 무엇을, 어디에 그릴 것인지 지정해준다. 이 render는 프로그램에 진입하는 main에 해당하는 역할을 한다.

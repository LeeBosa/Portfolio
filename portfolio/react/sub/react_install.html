React 설치하기

React를 시작하기 위한 두가지 방법이 있다. Create React App을 사용하여 새 어플리케이션을 제작하거나, 기존에 있던 어플리케이션에 React를 추가하는 것이다.

새 어플리케이션 만들기 (Creating a New Application)

Create React App은 새 React 싱글 페이지 어플리케이션 만들기를 시작하기 위한 최고의 방법이다. 최신 자바스크립트 기능을 사용할 수 있게 개발 환경을 세팅해주어, 좋은 개발자 경험을 사용하며, 프로덕션 앱을 위한 최적화도 진행한다. 기기에 Node >= 6 이상이 필요하다.

만약 npm 5.2.0+ 설치했다면, npx를 대신 사용할 수 있다.

Create React App은 backend 로직이나 데이터베이스를 제어할 수 없다. Create React App은 frontend 필드 파이프라인만 생성하기 때문에 backend를 원하는대로 사용할 수 있다. Create React App은 Babel이나 webpack같은 빌드 도구를 사용하나, 설정 없이도 동작한다.

프로덕션을 배포할 준비가 되었을 때, npm run build를 실행하면 build 폴더 안에 제작한 앱의 최적화된 빌드를 생성한다. Create React App README나 유저 가이드에서 더 자세한 사항을 볼 수 있다.

기존 앱에 React 추가하기 (Add React to an Existing Application)

React를 사용하기 위해 앱을 다시 작성할 필요는 없다.

개별 Widger같은 어플리케이션의 작은 부분부터 React를 추가하는 걸 권장하며 사용 사례에 맞게 동작하는 지 확인할 수 있다.

React는 빌드 파이프라인 없이도 사용이 가능하지만, 생산성을 높이기 위해 설정하는 걸 권장한다. 모던 빌드 파이프라인은 보통 다음과 같이 구성된다.

패키지 매니저에는 yarn이나 npm같은 게 있다. 패키지 매니저는 third-party 패키지의 방대한 생태계를 활용할 수 있게 하며, 쉽게 설치하고 업데이트 할 수 있게 한다.

번들러에는 webpack이나 browserify 같은 게 있다. 번들러는 코드를 모듈방식으로 작성할 수 있게 하고 이를 작은 패키지로 묶어서 로딩 시간을 최적화 할 수 있다.

컴파일러에는 Babel이 있다. 컴파일러는 모던 자바스크립트 코드를 작성해도 오래된 브라우저에서 동작할 수 있게 해준다.

React 설치하기 1 : 유형별 설치 방법

설치가 완료되면 프로덕션 환경에서 React의 빠른 버전을 사용하도록 프로덕션 빌드 프로세스를 설정하는 걸 강력히 권장한다.

또한, 프론트엔드 의존성을 관리하기 위해 yarn이나 npm을 사용하는 걸 권장한다.

npm vs yarn

npm은 자바스크립트 프로그래밍 언어를 위한 패키지 관리자이다. https://www.npmjs.com에서 공유되는 패키지들을 내려받아 활용할 수 있다. Node.js 설치 시 npm도 같이 설치되기 때문에 node.js를 설치했다면 따로 설치할 필요는 없다.

yarn은 npm 동작 방식과 비슷하지만, npm의 단점을 보완해 성능과 속도를 개선한 패키지 관리 도구이다.

npm은 여러 패키지를 설치할 경우 패키지가 완전히 설치될 때까지 기다렸다가 다른 패키지로 이동한다. 작업이 패키지별로 순차적으로 실행된다는 것이다. yarn은 이 작업들을 병렬로 설치하기 때문에 성능과 속도가 향상된다는 장점이 있지만, NPM과 비교했을 때 더 많은 용량을 차지한다.

개발 버전 VS 프로덕션 버전

기본적으로, React는 많은 도움이 되는 경고를 포함하고 있다. 이러한 경고들은 개발 단계에서 많은 도움이 된다.

그러나, 그러한 경고들이 React의 개발 버전을 크고 느리게 만들기 때문에 앱을 배포할 때는 프로덕션 버전을 사용해야 한다.

CDN 사용하기

npm을 사용한 클라이언트 패키지 관리를 하지 않으려는 경우, react와 react-dom npm 패키지는 CDN에서 호스트되는 umd 폴더에 단일 파일 배포를 제공한다.

react나 react-dom의 특정 버전을 불러오고 싶다면, 16을 원하는 버전의 숫자로 바꾸면 된다. 만약 Bower를 사용한다면, React는 react 패키지를 통해 사용할 수 있다.

crossorigin 속성이 왜 존재할까?

만약 React를 CDN으로 제공한다면, crossorigin 속성을 주는 걸 권장한다.

또한 사용중인 CDN이 Access-Control-Allow-Origin: * HTTP 헤더를 설정했는지 확인해야한다.

이 설정이 React 16과 그 이후 버전에서 더 나은 에러 처리 경험을 제공한다.

React 설치하기 2 : npm을 이용해 설치하기

1. 리액트 프로젝트에 사용할 폴더를 만든다.

2. Visual Studio Code에서 폴더 열기 후 방금 만든 폴더를 선택한다.

3. 터미널 메뉴에서 새 터미널을 연다.

4. npx create-react-app (프로젝트명)을 입력하고 엔터를 눌러 패키지를 설치한다.

5. 다시 폴더를 열고 들어가면 지정한 프로젝트명으로 폴더가 생성되어있다.

6. 폴더를 Visual Studio Code를 통해 열어보면 react 구동에 필요한 파일들이 설치되어있다.

7. 이제 터미널에서 npm start 명령어를 입력하여 react를 실행할 수 있다.
